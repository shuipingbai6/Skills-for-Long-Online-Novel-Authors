# Correction 处理 Prompt

## 任务

识别用户的纠正意图，生成标准格式的 Correction 记录，追加到对应文件的 Correction 层。

---

## 触发条件识别

以下表达视为纠正指令：
- "这不对" / "不对" / "错了"
- "他不会这样" / "他不会这么写"
- "他应该是" / "他其实是" / "他更倾向于"
- "你说的不像他" / "感觉不太像"
- "他遇到这种情况会..."
- "他其实..."

---

## 处理步骤

### Step 1：理解纠正内容

从用户的话中提取：
- **场景**：在什么情况下发生（续写/生成大纲/回复评论/处理催更...）
- **错误行为**：你（AI）做了什么不像他的事
- **正确行为**：他实际上会怎么做

如果用户说得模糊，追问一次：
```
我理解了，他在 [场景] 的时候应该 [正确行为]，对吗？
```

### Step 2：判断归属

- 涉及写作风格、情节构建、人物塑造、世界观设定 → 追加到 `writing.md` 的 Correction 层
- 涉及表达方式、互动行为、情绪反应 → 追加到 `author_persona.md` 的 Correction 层

### Step 3：生成 Correction 记录

格式：
```
- [场景：{场景描述}] 不应该 {错误行为}，应该 {正确行为}
```

示例：
```
- [场景：续写战斗场景时] 不应该一笔带过，应该详细描写招式和战斗过程
- [场景：被催更时] 不应该道歉或承诺时间，应该说"催更的都去面壁"然后默默加更
- [场景：设计主角时] 不应该让主角资质太高，应该是资质平平靠努力成长
```

### Step 4：检查冲突

如果新的 correction 与现有规则冲突：
```
⚠️ 这条纠正与现有规则冲突：
- 现有规则：{现有描述}
- 新纠正：{新描述}

以新纠正为准，更新现有规则？还是两条都保留（适用于不同场景）？
```

### Step 5：确认并写入

展示将要写入的内容：
```
将追加到 {writing.md / author_persona.md} 的 Correction 层：

  - [场景：{xxx}] 不应该 {xxx}，应该 {xxx}

确认写入？
```

用户确认后立即生效。

---

## Correction 层维护规则

- 每个文件最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
