# 骨架大纲生成 Prompt

## 任务

你将收到一章小说原文。你的任务是从中提取**骨架大纲**——只保留事件走向和情绪走向，省略所有风格相关的内容。

**核心目的**：骨架大纲将交给另一个 AI（加载了作者 Skill），让它根据大纲用自己的风格重写。因此大纲必须**不包含任何风格信息**，否则就变成了填空题，无法测试风格拟合度。

---

## 粒度标准

### ✅ 合适的粒度：骨架题

只给事件走向和情绪走向，让写作者自己决定怎么写。

```
第X章骨架：

场景：山顶修炼场
事件1：主角突破失败，受挫
事件2：师父出现，点拨
事件3：主角二次尝试，成功突破
情绪走向：挫败→顿悟→振奋
章节结尾：突破后感知到异象

写作者需要自己决定：
- 怎么描写突破失败（心理？身体反应？环境暗示？）
- 师父怎么点拨（对话？行为？回忆？）
- 成功突破怎么写（详细过程？一笔带过？）
- 异象是什么（视觉？听觉？第六感？）
```

### ❌ 太细的粒度：填空题

把对话、描写都写好了，写作者只需要填空，根本没在测试风格。

```
第X章大纲：

主角张三站在山顶修炼场中央，深吸一口气，灵力在体内运转。
他猛地冲击瓶颈，却感到一阵剧痛，突破失败了。
"可恶！"张三咬紧牙关，额头上冒出冷汗。
这时，师父李四从山路上走来，拍了拍张三的肩膀。
"你太急了，"李四说，"突破不是靠蛮力，要顺其自然。"
张三若有所思，重新盘膝坐下...
```

### ❌ 太粗的粒度：自由写作

只说"写一章"，写作者想写什么写什么，无法与原文对比。

```
写一章关于修炼的故事。
```

---

## 提取规则

1. **只提取事件节点**：发生了什么事，结果如何
2. **只标注情绪走向**：用箭头连接情绪变化（如"愤怒→冷静→坚定"）
3. **标注场景**：事件发生在哪里
4. **标注章节结尾状态**：本章结束时留下了什么悬念或转折
5. **省略所有风格内容**：
   - 不写对话内容（只写"发生了对话"或"通过对话得知了XX"）
   - 不写描写手法（不写"用环境暗示""用心理独白"）
   - 不写具体措辞（不写"他说了XXX"）
   - 不写修辞手法（不写"用了比喻""用了排比"）
6. **省略所有细节表达**：
   - 不写"他愤怒地握紧拳头"（这是具体表达），只写"他感到愤怒"（这是情绪）
   - 不写"雨打在窗户上"（这是环境描写），只写"场景：雨夜"（这是场景设定）

---

## 输出格式

对每一章，输出以下格式的大纲：

```
### 第{N}章骨架

**场景**：{场景描述，1句话}

**事件链**：
1. {事件1} → {结果1}
2. {事件2} → {结果2}
3. {事件3} → {结果3}
...

**情绪走向**：{情绪A} → {情绪B} → {情绪C}

**章节结尾**：{结尾状态，如"留下悬念：XXX"/"转折：XXX"/"新信息揭露：XXX"}

**关键约束**（如有）：
- {必须遵守的情节约束，如"主角必须受伤"/"必须有人牺牲"}
- {不能违反的设定约束，如"此时主角尚不知晓某信息"}
```

---

## 多章处理

如果收到多章原文，对每章分别生成骨架大纲，章节之间用分隔线隔开：

```
### 第{N}章骨架
{...}

---

### 第{M}章骨架
{...}

---

### 第{P}章骨架
{...}
```

---

## 注意事项

1. 骨架大纲的目标读者是另一个 AI，它将根据这个大纲和作者 Skill 来写作
2. 大纲的详细程度应该让写作者知道"写什么"，但不知道"怎么写"
3. 如果原文中有些事件无法简化为骨架（如纯氛围描写章节），标注为"氛围章节：{核心氛围}"
4. 保持客观：大纲中不要加入你的评价或分析
