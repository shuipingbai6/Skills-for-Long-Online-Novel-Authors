# 重整式 Skill 更新 Prompt

## 任务

你将收到：
1. 现有的 `writing.md` 内容
2. 现有的 `author_persona.md` 内容
3. 新采样的小说章节内容

你的任务是**重写** writing.md 和 author_persona.md，将新章节的观察融入已有结论，输出更精确但同等精炼的完整文件。

**核心原则：融入而非追加，保持精炼。每轮重写，而非在末尾拼贴新内容。**

类比泰勒级数：从 2 阶升到 3 阶时，多项式仍然是多项式，不是在 2 阶多项式后面拼一个 3 阶项。

---

## Step 1：提取新观察

从新采样的章节中提取风格特征，按以下维度归类：

| 维度 | 归入 |
|------|------|
| 叙事视角、叙事节奏、叙事手法、时间线处理 | → writing.md / 叙事风格 |
| 开篇套路、高潮设计、收尾习惯、情节推进、爽点设计 | → writing.md / 情节构建 |
| 主角设定、配角塑造、反派设计、人物关系、人物成长 | → writing.md / 人物塑造 |
| 力量体系、世界背景、规则设定、经济系统、社会结构 | → writing.md / 世界观设定 |
| 对话比例、对话风格、对话功能、对话长度、口头禅 | → writing.md / 对话风格 |
| 环境描写、心理描写、战斗描写、情感描写、感官描写 | → writing.md / 描写风格 |
| 表达风格、口头禅与高频词、说话方式 | → author_persona.md / Layer 2 |
| 创作理念、对读者态度、对作品态度 | → author_persona.md / Layer 3 |
| 互动行为（评论回复、催更回应等） | → author_persona.md / Layer 4 |

---

## Step 2：判断关系并融合

对比新观察与已有 skill 内容，按三种关系处理：

### 情况一：一致 → 强化

新章节的风格特征与 skill 已有描述一致时：

- **强化置信度**，不重复写入
- 可在该规则后标注验证信息，如"（已验证：前5章+第50章一致）"
- 如果已有规则已足够精确，不需要任何修改

示例：
```
已有：主角对话风格口语化，常用短句
新观察：第 50 章对话仍然口语化短句
→ 不修改该规则，或标注"（已验证至第50章）"
```

### 情况二：补充 → 融合

新章节提供了 skill 中未覆盖的风格细节时：

- 将新细节融入对应维度的已有描述，合并为更完整的规则
- 保持条件-行为格式，不变成描述性标签
- 融合后的规则必须比原来更精确，而非更长

示例：
```
已有：战斗描写详细，注重招式
新观察：第 80 章战斗中大量插入心理活动
→ 融合为："战斗描写详细，注重招式；重要战斗中每3个动作插入一次心理闪念"
```

### 情况三：矛盾 → 标注演变

新章节的风格特征与 skill 已有描述矛盾时：

- 标注风格演变，保留两者并注明阶段
- 使用"前期……；后期……"格式
- 不要删除已有规则，而是扩展为分阶段描述

示例：
```
已有：叙事节奏快，每章必有冲突推进
新观察：第 200 章开始节奏明显放缓，大量日常描写
→ 修改为："前期叙事节奏快，每章必有冲突推进；后期节奏放缓，允许连续2-3章日常铺陈"
```

## Step 3：长度控制

重写后的文件必须保持在预设上限内：

| 文件 | 上限 |
|------|------|
| writing.md | 6000 字 |
| author_persona.md | 3000 字 |

如果重写后超过上限，按以下优先级压缩（从低到高，优先压缩低优先级内容）：

1. **风格示例段落**：可精简或替换为更代表性的摘录
2. **描述性内容**：可压缩为更精炼的表述
3. **重复或冗余的规则**：可合并为一条更完整的规则
4. **条件-行为格式规则**：尽量保留，但可精简措辞
5. **Layer 0 核心规则**：**绝不压缩**

压缩原则：
- 合并同类规则（如两条关于对话的规则可合并为一条更完整的）
- 删除被验证规则覆盖的占位内容（如"原材料不足"标注被新信息替代后删除）
- 精简示例段落（保留最具代表性的，删除冗余的）

## Step 4：输出完整重写后的文件

输出两个**完整的**重写后文件，不是增量 patch。

```
=== writing.md（重写后完整内容）===

{完整的重写后 writing.md 内容}

=== author_persona.md（重写后完整内容）===

{完整的重写后 author_persona.md 内容}
```

## Step 5：生成更新摘要

向用户展示本轮更新的核心变化：

```
本轮更新摘要：
- 采样章节：第{N}章、第{M}章、第{P}章...
- 更新维度：
  - writing.md：{列出有变化的维度，标注一致/补充/矛盾}
  - author_persona.md：{列出有变化的层级，标注一致/补充/矛盾}
- 长度变化：writing.md {旧字数}→{新字数}，author_persona.md {旧字数}→{新字数}
- 版本将从 {vN} 升级到 {vN+1}
```

---

## 重要注意事项

1. **绝不追加**：不要在已有内容后面拼接新段落，而是将新信息融入已有结构
2. **保持格式**：重写后的文件必须保持原有的 Markdown 结构和标题层级
3. **保持精炼**：每轮重写后的文件长度应与之前相当或更短，不应持续增长
4. **保留占位**：如果某个维度仍然信息不足，保留"（原材料不足，建议追加相关小说章节）"标注
5. **保留 Correction 记录**：不要删除已有的 Correction 记录
6. **条件-行为格式**：所有规则必须是"在什么条件下必须/禁止做什么"的格式，禁止描述性标签
