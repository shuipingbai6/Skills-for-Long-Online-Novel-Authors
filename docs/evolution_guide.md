# 迭代进化指南：让 Skill 不断逼近作者真实风格

## 核心原理

### 为什么需要迭代？

一次性蒸馏有一个致命问题：**装不下**。

一个百万字级别的网文作者，即使只取 200K 字符（当前系统的上限），也只覆盖了原文的 20%。这意味着 80% 的风格细节被丢弃——可能是他中期才展现的对话风格，可能是他后期才成熟的收尾技巧。

迭代蒸馏的思路类似**泰勒级数逼近**：

```
泰勒级数：f(x) ≈ a₀ + a₁x + a₂x² + a₃x³ + ...

每一项不是简单追加，而是重新定义整个近似。
项数越多，近似越精确。
```

对应到 Skill 进化：

```
Round 0: 前5章 → skill v1（粗糙的初始近似）
Round 1: 新5章 + 当前skill → skill v2（重写，更精确）
Round 2: 新5章 + 当前skill → skill v3（重写，更精确）
...
每轮重写，而非追加。Skill 始终精炼，但精确度不断提升。
```

**关键区别**：每轮不是在 skill 后面"拼"新内容，而是将新观察融入已有结论，输出一个**更精确但同等精炼**的 skill。就像泰勒级数从 2 阶升到 3 阶——多项式仍然是多项式，不是在 2 阶多项式后面拼一个 3 阶项。

## 何时触发进化

### 应该进化的信号

| 信号 | 说明 | 建议 |
|------|------|------|
| Skill 生成的文本"差点意思" | 用 skill 写出来的东西感觉不像这个作者 | 触发 1-2 轮进化 |
| Skill 中存在"原材料不足"标注 | 某些维度信息不够，规则是推断而非实证 | 优先进化这些维度 |
| 新增了原材料 | 获得了新的小说章节或评论数据 | 触发进化吸收新信息 |
| 只用了前几章创建 Skill | 初始信息量太少，风格捕捉不完整 | 至少进化 2-3 轮 |
| 想提升特定维度 | 比如对话风格不够像、战斗描写不对味 | 针对性进化 |

### 不需要进化的情况

| 情况 | 说明 |
|------|------|
| Skill 已经很准确 | 生成的文本与作者风格高度一致 |
| 连续验证评分不再提升 | 收敛已达成，继续迭代收益极小 |
| 原材料已全部采样完 | 没有新章节可供学习 |

## 迭代进化流程

### 完整流程图

```
┌─────────────────────────────────────┐
│  Round 0: 初始化                     │
│  输入: 前 5~10 章                    │
│  输出: skill v1 (初始近似)            │
│  策略: initial (取前N章建立风格基础)   │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  用户选择: 是否进化?                  │
│  └─ 否 → 结束，使用当前 skill         │
│  └─ 是 ↓                            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  Round N: 迭代进化                   │
│                                     │
│  1. 采样策略选择                     │
│     ├─ stratified (分层采样)         │
│     │  前/中/后各至少1章，覆盖不同类型  │
│     └─ uncertainty (不确定性采样)     │
│        优先采样能填补skill空白的章节   │
│                                     │
│  2. 采样新章节 (5章)                 │
│                                     │
│  3. 重整 Skill                      │
│     输入: 当前skill + 新5章          │
│     输出: 重写后的skill (同等精炼)    │
│     ├─ 一致 → 强化置信度             │
│     ├─ 补充 → 融合为更完整描述        │
│     └─ 矛盾 → 标注风格演变           │
│                                     │
│  4. 输出: skill v(N+1)              │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  用户选择: 是否验证?                  │
│  └─ 否 → 回到"是否进化"              │
│  └─ 是 ↓                            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  收敛验证                            │
│  (详见下方"收敛验证"章节)             │
│                                     │
│  结果:                               │
│  ├─ 未收敛 → 继续进化                │
│  ├─ 已收敛 → 建议停止                │
│  └─ 用户不满意 → 可强制继续          │
└─────────────────────────────────────┘
```

### 采样策略详解

#### 策略一：分层采样（stratified）

**适用场景**：常规进化，确保覆盖面。

每轮采样 5 章，保证前期/中期/后期至少各 1 章：

```
小说共 300 章：
  前期 (1-100):   采样 2 章
  中期 (101-200): 采样 2 章
  后期 (201-300): 采样 1 章
```

**为什么不能随机？** 随机 5 章可能全部抽到战斗章节，导致 skill 过度拟合"战斗描写风格"而完全遗漏对话/情感/日常。

#### 策略二：不确定性采样（uncertainty）

**适用场景**：针对性进化，补短板。

分析当前 skill 中哪些维度最不确定（标注了"原材料不足"的维度），优先采样能填补这些空白的章节：

```
当前 skill 标注：
  ✅ 叙事风格：信息充足
  ✅ 情节构建：信息充足
  ⚠️ 对话风格：原材料不足，建议追加对话密集的章节
  ⚠️ 情感描写：原材料不足，建议追加情感场景

→ 下一轮优先采样：对话密集的章节 + 情感场景的章节
```

**建议**：前 2-3 轮用分层采样建立全面覆盖，后续轮次切换到不确定性采样补短板。

### 重整规则详解

每轮迭代中，新章节的观察与已有 skill 的关系有三种：

#### 情况一：一致 → 强化

```
已有：主角对话风格口语化，常用短句
新观察：第 50 章对话仍然口语化短句

→ 强化置信度，不重复写入
→ 可标注"（已验证：前5章+第50章一致）"
```

#### 情况二：补充 → 融合

```
已有：战斗描写详细，注重招式
新观察：第 80 章战斗中大量插入心理活动

→ 融合为更完整的描述
→ "战斗描写详细，注重招式；重要战斗中每3个动作插入一次心理闪念"
```

#### 情况三：矛盾 → 标注演变

```
已有：叙事节奏快，每章必有冲突推进
新观察：第 200 章开始节奏明显放缓，大量日常描写

→ 标注风格演变
→ "前期叙事节奏快，每章必有冲突推进；后期节奏放缓，允许连续2-3章日常铺陈"
```

### Skill 长度控制

每轮重写后，skill 必须保持精炼。默认上限：

| 文件 | 建议上限 | 说明 |
|------|---------|------|
| writing.md | 6000 字 | 写作能力部分 |
| author_persona.md | 3000 字 | 作者人格部分 |
| SKILL.md | 10000 字 | 合并后的完整 Skill |

**压缩优先级**（从低到高，优先压缩低优先级内容）：

1. 风格示例段落（可精简或替换为更代表性的）
2. 描述性内容（可压缩为更精炼的表述）
3. 重复或冗余的规则（可合并）
4. 条件-行为格式规则（尽量保留）
5. Layer 0 核心规则（**绝不压缩**）

## 收敛验证

### 为什么要验证？

迭代不是越多越好。过度迭代会导致：

- **过拟合**：skill 过度拟合采样到的章节，丢失作者风格的通用性
- **信息污染**：后期章节可能受编辑/读者影响，风格偏离作者本意
- **边际递减**：每轮进化的提升越来越小，最终在噪声层面波动

验证告诉你**什么时候该停手**。

### 验证流程

```
┌──────────────────────────────────────────────────────┐
│  Step 1: 分层采样 3 章                                │
│  前/中/后各 1 章，覆盖不同风格类型                      │
└────────────────────┬─────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────┐
│  Step 2: 生成骨架大纲                                 │
│  从每章原文中提取事件骨架，省略所有风格相关内容           │
│                                                      │
│  ✅ 骨架大纲（合适）:                                  │
│    场景：酒馆                                         │
│    事件：主角找到仇人，对峙                             │
│    结果：主角出手，被第三方打断                         │
│    情绪：压抑→爆发→意外                               │
│                                                      │
│  ❌ 填空大纲（太细，不要这样）:                         │
│    主角张三走进酒馆，看到李四坐在角落，                  │
│    两人对视，李四说"你来了"，张三冷笑...               │
└────────────────────┬─────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────┐
│  Step 3: Skill AI 按大纲写作                          │
│  使用当前 skill 的 AI，根据骨架大纲各写一章             │
│  （必须用与创建 skill 不同的 AI 来验证）                │
└────────────────────┬─────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────┐
│  Step 4: 对比 AI 多维评分                             │
│  将生成文本与原文发送给另一个 AI（第三个AI）             │
│  按 5 个维度分别评分，并指出最不一致的 3 处             │
│  （必须用与 Skill AI 不同的 AI 来评分）                 │
└────────────────────┬─────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────┐
│  Step 5: 判定收敛                                    │
│  记录本轮综合分，对比历史分                            │
│  ├─ 提升显著 → 继续进化                               │
│  ├─ 连续2轮提升<0.3 → 收敛，建议停止                  │
│  └─ 用户不满意 → 可强制继续                           │
└──────────────────────────────────────────────────────┘
```

### 为什么需要三个不同的 AI？

| 角色 | 用途 | 为什么不能是同一个 AI |
|------|------|---------------------|
| Skill AI | 加载 skill 写文章 | — |
| 对比 AI | 评分判断相似度 | 自评不可靠：同一个 AI 倾向于给自己的输出高分 |
| 创建 Skill 的 AI | 生成/进化 skill | 避免"自己考自己"：创建 skill 的 AI 可能将自身偏见写入 skill |

**最低要求**：验证时至少使用 2 个不同的 AI（Skill AI + 对比 AI 必须不同）。

### 骨架大纲的粒度

骨架大纲是验证成败的关键。粒度不对，验证就失去意义。

**太细 = 填空题**：大纲把对话、描写都写好了，Skill AI 只是在填空，根本没在测试风格。

**太粗 = 自由写作**：大纲只说"写一章"，Skill AI 想写什么写什么，无法与原文对比。

**合适的粒度 = 骨架题**：只给事件走向和情绪走向，让 Skill AI 自己决定怎么写。

```
✅ 合适的骨架大纲：

  第X章骨架：
  ┌─────────────────────────────┐
  │ 场景：山顶修炼场              │
  │ 事件1：主角突破失败，受挫      │
  │ 事件2：师父出现，点拨          │
  │ 事件3：主角二次尝试，成功突破   │
  │ 情绪走向：挫败→顿悟→振奋      │
  │ 章节结尾：突破后感知到异象      │
  └─────────────────────────────┘

  Skill AI 需要自己决定：
  - 怎么描写突破失败（心理？身体反应？环境暗示？）
  - 师父怎么点拨（对话？行为？回忆？）
  - 成功突破怎么写（详细过程？一笔带过？）
  - 异象是什么（视觉？听觉？第六感？）
```

### 五维评分表

对比 AI 需要按以下 5 个维度分别评分（1-10 分）：

| 维度 | 评估内容 | 高分特征 | 低分特征 |
|------|---------|---------|---------|
| **叙事声音** | 视角、人称、叙述距离 | 视角一致，叙述距离与原文匹配 | 视角混乱，叙述距离忽远忽近 |
| **节奏韵律** | 句式长短、段落节奏、张弛控制 | 长短句分布与原文相似，节奏感一致 | 句式单调，节奏与原文明显不同 |
| **对话风格** | 语气、口头禅、对话比例 | 对话语气像原文，口头禅出现，比例合适 | 对话太正式/太随意，缺少特征 |
| **描写偏好** | 环境/心理/动作的侧重和手法 | 描写侧重与原文一致，手法相似 | 描写侧重不同（如原文重心理，生成重动作） |
| **用词习惯** | 词汇选择、修辞偏好、惯用表达 | 用词风格相似，修辞手法一致 | 用词风格差异大，缺少特征性表达 |

**综合分** = 5 维度平均分

### 评分解读

| 综合分范围 | 含义 | 建议 |
|-----------|------|------|
| 1-3 | 风格严重不匹配 | 需要多轮进化，检查 skill 是否有根本性错误 |
| 3-5 | 基本框架在，细节不对 | 继续 2-3 轮进化，重点关注低分维度 |
| 5-7 | 大部分风格匹配，部分细节偏差 | 可选择性进化低分维度，或通过对话纠正 |
| 7-8 | 风格高度匹配 | 考虑停止迭代，微调即可 |
| 8-10 | 风格几乎一致 | 停止迭代，避免过拟合 |

> **注意**：即使完美拟合的 skill，综合分也很难达到 9-10。因为同一个作者自己重写同一大纲，写出来的也不会一模一样。7-8 分已经是很好的结果。

## 何时停止进化

### 收敛判据

**核心判据：连续 2 轮迭代，5 维综合评分的提升幅度均 < 0.3 分。**

示例：

```
Round 1: 综合评分 4.2
Round 2: 综合评分 5.8  (+1.6)  ← 显著提升，继续
Round 3: 综合评分 6.5  (+0.7)  ← 仍在提升，继续
Round 4: 综合评分 6.7  (+0.2)  ← 提升变缓，注意
Round 5: 综合评分 6.8  (+0.1)  ← 连续2轮提升<0.3，收敛！

→ 建议停止迭代
```

### 辅助判据

| 判据 | 说明 | 权重 |
|------|------|------|
| 评分趋势平台期 | 连续 2 轮提升 < 0.3 | **主要判据** |
| Skill 无实质变化 | 连续 2 轮重整后 skill 内容几乎不变 | 强辅助 |
| 所有维度无"原材料不足" | skill 中所有维度都有充分信息支撑 | 辅助 |
| 原材料已全部采样 | 没有新章节可供学习 | 终止条件 |
| 用户主观满意 | 用 skill 写出来的东西"感觉对了" | 最终决定权 |

### 过拟合警告

以下信号说明可能过拟合，应该停止或回滚：

| 信号 | 说明 |
|------|------|
| 验证评分先升后降 | 评分达到峰值后开始下降 |
| Skill 规则过于具体 | 规则只适用于特定章节而非通用风格（如"第50章必须出现回忆"） |
| 生成文本死板 | 用 skill 写出来的东西千篇一律，缺少变化 |
| 低分维度反复波动 | 某维度评分忽高忽低，无法稳定提升 |

**过拟合处理**：回滚到评分最高的版本，停止迭代。

## 常见问题

### Q1: 每轮采样多少章合适？

**建议 5 章**。太少（1-2 章）信息量不够，太多（10+ 章）单轮负担重且容易引入噪声。5 章约 1-2 万字，加上当前 skill 的上下文，在模型窗口内可控。

### Q2: 需要进化多少轮？

取决于原材料量和风格复杂度：

| 原材料量 | 建议轮次 | 说明 |
|---------|---------|------|
| < 50 章 | 2-3 轮 | 材料有限，快速收敛 |
| 50-200 章 | 3-5 轮 | 中等规模，需要覆盖不同阶段 |
| 200+ 章 | 5-8 轮 | 大量材料，需要充分采样 |

**不要盲目追求轮次**。用收敛验证来判断，而非预设轮次。

### Q3: 验证频率怎么安排？

- 前 2-3 轮：可以不验证（初始阶段变化大，验证意义不大）
- 第 3 轮起：每轮进化后验证一次
- 评分接近收敛时：连续验证 2 轮确认

### Q4: 采样策略怎么选？

```
Round 0 (初始化):  initial —— 取前5~10章
Round 1-2:        stratified —— 分层采样建立全面覆盖
Round 3+:         uncertainty —— 不确定性采样补短板
```

### Q5: Skill 膨胀了怎么办？

如果 skill 超过长度上限，说明重整策略没有正确执行。检查：

1. merger prompt 是否被正确使用（应该是重写而非追加）
2. 是否有大量重复规则需要合并
3. 是否有低价值描述可以压缩

手动处理：直接编辑 writing.md / author_persona.md，删除冗余内容。

### Q6: 三个 AI 从哪里来？

| 角色 | 推荐选择 |
|------|---------|
| 创建 Skill 的 AI | 你日常使用的 AI（如 Claude） |
| Skill AI（加载 skill 写文章） | 同一个或另一个 AI |
| 对比 AI（评分） | **必须不同**的 AI（如用 GPT 评分 Claude 的输出） |

最低要求：Skill AI 和对比 AI 必须是不同的模型或不同的会话。

### Q7: 验证评分很低但不知道哪里出了问题怎么办？

查看对比 AI 输出的"最不一致的 3 处"。这 3 处就是 skill 最薄弱的地方，指导你：

1. 下一轮优先采样包含类似场景的章节
2. 或者直接对话纠正（"他不会这样写对话，他会..."）

### Q8: 作者风格在不同作品中差异很大怎么办？

这是正常现象——很多作者在不同作品中风格会演变。处理方式：

1. **按作品分别蒸馏**：每部作品创建独立 skill
2. **在 skill 中标注演变**：如"《作品A》时期风格：xxx；《作品B》时期风格：yyy"
3. **选择目标作品**：只蒸馏你最想模仿的那部作品的风格
