## 系统概述

本系统将网文作者的创作风格、叙事技巧、世界观构建能力蒸馏成可复用的 AI Skill，实现风格续写、创作辅助、风格分析等功能。

### 核心能力

| 能力 | 说明 |
|------|------|
| 写作风格提取 | 从小说文本中提取叙事风格、情节构建、人物塑造、世界观设定 |
| 作者人格提取 | 从评论回复、社交媒体中提取作者的人格特征 |
| 标签体系 | 网文作者专属标签（写作风格、更新习惯、人格标签、题材标签） |
| 持续进化 | 支持追加文件、对话纠正 |
| 风格续写 | 使用生成的 Skill 进行风格续写 |

### 详细步骤说明

#### Step 1: 触发 Skill

**触发方式**：
```
/create-author
```
或直接说：
- "帮我创建一个作者 skill"
- "我想蒸馏一个作者"
- "新建作者"
- "给我做一个 XX 的 skill"

#### Step 2: 基础信息录入

系统会依次询问 3 个问题：

**问题 1: 笔名/代号**（必填）
```
这位作者怎么称呼？（笔名、昵称或代号都行）

例：fei-tian-yu
```

**问题 2: 基本信息**（可选）
```
用一句话描述他的基本信息——平台、等级、代表作、性别，想到什么写什么。

例：起点 LV5 玄幻作家 男 代表作《万古神帝》
```

**问题 3: 性格画像**（可选）
```
用一句话描述他的性格——写作风格、更新习惯、人格特点、题材、你对他的印象。

例：水文大师 日更党 宠粉狂魔 玄幻 擅长装逼打脸
```

#### Step 3: 原材料导入

系统会展示导入选项：
```
原材料怎么提供？

  [A] 上传文件
      txt / epub 小说文件
      评论导出文件（起点/晋江/番茄）
      微博/公众号文章截图或导出

  [B] 手动导入
      提供小说文件路径
      提供评论链接（需手动采集）

  [C] 直接粘贴内容
      把文字复制进来

可以混用，也可以跳过（仅凭手动信息生成）。
```

**文件存放位置**：
- 小说文件 → `knowledge/novels/`
- 评论数据 → `knowledge/comments/`
- 社交媒体 → `knowledge/social/`

#### Step 4: 分析原材料

系统会按两条线路并行分析：

**线路 A (Writing Skill)**：
- 叙事风格：视角、节奏、手法、时间线处理
- 情节构建：开篇套路、高潮设计、收尾习惯、爽点设计
- 人物塑造：主角设定、配角塑造、反派设计、人物关系
- 世界观设定：力量体系、世界背景、规则设定、经济系统
- 对话风格：对话比例、风格、功能、长度
- 描写风格：环境、心理、战斗、情感、感官描写

**线路 B (Author Persona)**：
- 表达风格：口头禅、高频词、句式特征、emoji 使用
- 创作理念：写作动机、对读者态度、对作品态度
- 互动行为：对评论、催更、质疑、建议、打赏的回应
- 边界与雷区：抵触的事情、拒绝的场景、回避的话题

#### Step 5: 生成并预览

系统会展示摘要供确认：
```
Writing Skill 摘要：
  - 叙事风格：第三人称限制视角，快节奏叙事，每章必有冲突
  - 情节构建：退婚流开篇，装逼打脸套路，每章结尾留悬念
  - 人物塑造：主角资质平平靠金手指成长，反派智商在线但必败
  ...

Author Persona 摘要：
  - 核心性格：评论回复犀利，爱吐槽读者，嘴上不饶人但心里关心
  - 表达风格：常用"催更的都去面壁"，句式简短，喜欢用感叹号
  - 互动行为：积极回复评论，经常加更，重视读者反馈
  ...

确认生成？还是需要调整？
```

#### Step 6: 写入文件

确认后，系统会创建以下文件：

```
authors/{slug}/
├── SKILL.md          # 完整 Skill（可直接调用）
├── writing.md        # 写作能力
├── author_persona.md # 作者人格
├── meta.json         # 元数据
├── versions/         # 历史版本存档
└── knowledge/        # 该作者的原始材料
    ├── novels/       # 小说文件（从 knowledge/ 复制）
    ├── comments/     # 评论数据
    └── social/       # 社交媒体内容
```

**meta.json 结构**：
```json
{
  "name": "飞天鱼",
  "slug": "fei-tian-yu",
  "created_at": "2026-04-11T10:00:00Z",
  "updated_at": "2026-04-11T10:00:00Z",
  "version": "v1",
  "profile": {
    "platform": "起点",
    "level": "LV5",
    "masterpiece": "万古神帝",
    "gender": "男"
  },
  "tags": {
    "writing_style": ["水文大师", "爽文流"],
    "update_habit": ["日更党"],
    "personality": ["宠粉狂魔"],
    "genre": ["玄幻"]
  },
  "impression": "擅长装逼打脸",
  "knowledge_sources": ["novel_1.txt", "comments_1.txt"],
  "corrections_count": 0
}
```

#### Step 7: 完成

生成完成后，可以通过以下触发词使用：

| 触发词 | 说明 |
|--------|------|
| `/{slug}` | 完整版（写作能力 + 作者人格） |
| `/{slug}-writing` | 仅写作能力 |
| `/{slug}-persona` | 仅作者人格 |

## 进化机制

### ⚠️ 重要说明

**进化不是自动的**，需要用户主动触发。系统支持两种进化方式：

| 进化方式 | 触发条件 | 说明 |
|----------|----------|------|
| 追加文件 | 说"我有新文件"或"追加" | 提供新的小说/评论/社交媒体内容 |
| 对话纠正 | 说"这不对"或"他不会这样" | 直接纠正 Skill 的行为 |

### 方式一：追加文件

当用户说"我有新文件"或"追加"时，系统会：

```
用户: 我有新文件

系统: 请提供新的小说文件或评论数据

用户: [上传 novel_2.txt]

系统: 正在解析新文件...
✅ 解析完成：识别 50 章，总字数 20 万字

正在分析增量内容...

本次更新摘要：
- writing.md：追加了 3 条新信息（新的情节套路、人物关系）
- author_persona.md：追加了 2 条新信息（新的口头禅、互动方式）

版本将从 v1 升级到 v2。
确认应用更新？

用户: 确认

系统: ✅ Skill 已更新到 v2
```

**流程**：
1. 用户说"我有新文件"
2. 系统请求上传文件
3. 系统解析并分析增量内容
4. 展示更新摘要
5. 用户确认后写入

### 方式二：对话纠正

当用户说"这不对"、"他不会这样"、"他应该是"时，系统会：

```
用户: 他不会这样写，他遇到装逼场景时会详细描写心理活动

系统: 我理解了，他在 [装逼场景] 的时候应该 [详细描写心理活动]，对吗？

用户: 对

系统: 将追加到 writing.md 的 Correction 层：

  - [场景：装逼场景时] 不应该一笔带过，应该详细描写主角的心理活动和装逼过程

确认写入？

用户: 确认

系统: ✅ Correction 已写入，Skill 已更新
```

**流程**：
1. 用户表达纠正意图
2. 系统理解并确认纠正内容
3. 判断归属（Writing 或 Persona）
4. 生成 Correction 记录
5. 用户确认后写入

### 方式三：命令更新

使用命令直接更新指定作者：

```
/update-author fei-tian-yu
```

系统会引导用户完成更新流程。

### 版本管理

每次更新都会：
1. 自动存档当前版本到 `authors/{slug}/versions/`
2. 版本号自动递增（v1 → v2 → v3...）
3. 最多保留 10 个历史版本

**回滚命令**：
```
/author-rollback fei-tian-yu v1
```

## 数据收集指南

### 1. 小说文件收集

#### txt 文件

**推荐格式**：
- UTF-8 编码
- 标准章节格式（如"第一章 xxx"、"第1章 xxx"）
- 至少 50 万字（玄幻作者）/ 30 万字（都市作者）

**支持的章节格式**：
- 中文：`第X章`、`第X节`、`第X卷`、`第X篇`
- 英文：`Chapter X`、`Part X`、`Section X`
- 数字：`1.`、`1.1`、`一、`、`【一】`

**存放位置**：`knowledge/novels/`

#### epub 文件

**推荐**：
- 标准 epub 2.0/3.0 格式
- 包含完整的章节结构

**存放位置**：`knowledge/novels/`

### 2. 评论数据收集

#### 起点中文网

**导出方式**：
1. 打开作品页面
2. 滚动到评论区
3. 手动复制评论内容
4. 保存为 txt 文件

**格式示例**：
```
2024-01-01 12:00
用户A：这章太水了
作者回复：催更的都去面壁

2024-01-01 13:00
用户B：好看！求加更！
作者回复：明天爆更！
```

**存放位置**：`knowledge/comments/`

#### 晋江文学城

**格式示例**：
```
[用户A] 这章写得真好！
[作者] 谢谢支持~

[用户B] 求更新！
[作者] 在写了在写了
```

**存放位置**：`knowledge/comments/`

#### 番茄小说

**格式示例**：
```
1楼
用户A：好看！

2楼
用户B：催更！
作者回复：明天更新
```

**存放位置**：`knowledge/comments/`

### 3. 社交媒体内容收集

#### 微博

**导出方式**：
1. 打开作者微博主页
2. 手动复制微博内容
3. 保存为 txt 文件

**格式示例**：
```
2024-01-01
今天状态不错，爆更三章！

2024-01-02
催更的都去面壁，我在写了！
```

**存放位置**：`knowledge/social/`

#### 微信公众号

**导出方式**：
1. 打开公众号文章
2. 复制文章内容或保存为 HTML
3. 保存为 txt 或 html 文件

**存放位置**：`knowledge/social/`

### 4. 数据质量要求

#### 玄幻作者

| 数据类型 | 最低要求 | 推荐要求 |
|----------|----------|----------|
| 小说正文 | 50 万字 | 100 万字+ |
| 评论回复 | 100 条 | 500 条+ |
| 社交媒体 | 10 条 | 50 条+ |
| 章节数 | 100 章 | 200 章+ |

#### 都市作者

| 数据类型 | 最低要求 | 推荐要求 |
|----------|----------|----------|
| 小说正文 | 30 万字 | 50 万字+ |
| 评论回复 | 50 条 | 200 条+ |
| 章节数 | 60 章 | 100 章+ |

## 常见问题

### Q1: 生成的 Skill 不够准确怎么办？

**A**: 可以通过以下方式改进：
1. **追加更多材料**：提供更多小说章节、评论回复
2. **对话纠正**：直接说"他不会这样"，系统会记录纠正
3. **手动调整**：直接编辑 `writing.md` 和 `author_persona.md`

### Q2: 支持哪些小说格式？

**A**: 
- txt（推荐 UTF-8 编码）
- epub（标准 2.0/3.0 格式）

### Q3: 评论数据从哪里获取？

**A**: 
- 起点/晋江/番茄等平台的评论区
- 手动复制或截图
- 第三方工具导出

### Q4: 生成的 Skill 可以用于商业用途吗？

**A**: 
**不可以**。本工具仅供个人学习和研究使用，不得用于商业用途。详见法律免责声明。

### Q5: 如何回滚到历史版本？

**A**: 
```
/author-rollback {slug} {version}
```
例如：`/author-rollback fei-tian-yu v1`

### Q6: 如何删除作者 Skill？

**A**: 
```
/delete-author {slug}
```
例如：`/delete-author fei-tian-yu`

### Q7: 数据存放在哪里？

**A**: 
- **待处理的原始材料**：`knowledge/` 目录
- **生成的 Skill**：`authors/{slug}/` 目录
- **每个作者的材料**：`authors/{slug}/knowledge/` 目录

### Q8: 如何更新已有的 Skill？

**A**: 
1. **追加文件**：说"我有新文件"
2. **对话纠正**：说"他不会这样"
3. **命令更新**：`/update-author {slug}`
